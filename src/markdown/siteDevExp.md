本文介绍了 Web 网站部署过程中的个人经验和遇到的问题。

# 1.域名注册与备案

申请不到，符合管局规定的可以试试。

域名可以直接在云服务商处购买，一般没有特殊含义的域名，都有试用期（一年）优惠，价格低廉。
国内云服务商（阿里云 / 腾讯云等）仅为已备案域名提供解析与接入服务；
可以直接在购买域名的服务商处完成域名预备案，然后由服务商向管局备案。

一般关系流程：
**备案者 ↔（预审） 云服务商 ↔ 通信管理局**

不同管局对备案的要求不一致，以下是 **新疆管局对个人备案** 的要求（摘自阿里云）：
- 非本省身份证不能进行个人备案。
- 个人未满 18 周岁不允许进行 ICP 备案。
- 已备案成功的网站开通后需在网站底部悬挂网站 ICP 备案号，并链接至工信部备案平台。
- 自 2018 年 01 月 01 日 0 点起，对提交的网站 ICP 备案申请，进行网站备案域名核验。
- 域名有效期需大于或等于 3 个月。
- 域名持有者需与主办人名称一致。
- 个人网站名称需遵循官方命名规范。

## 不备案域名解析的其他手段
缺点很多，而且需要特殊渠道，后续可能被墙，不推荐。
1. **使用境外服务器**
   域名解析到香港、新加坡、美国等境外服务器。
2. **境外 CDN / 对象存储**
   使用 Cloudflare 等海外 CDN，或境外对象存储托管网站。




# 2.云服务器购置与Debian系统基础配置
直接在云服务商处购买服务器并配置即可。  
防火墙相关配置（以我的百度云为例）:  
打开云服务器控制台。  
找到“安全组”相关模块。  
新增一个安全组，修改出站和入站规则。  
确认ssh或其他和服务器通信的方式，其配置已经修改，否则使用默认端口可能被安全组拦截。
绑定此安全组到服务器。  


# 3.1Panel部署与基础服务配置
1Panel是服务器可视化面板，提供了可视化的一些便捷操作，直接按照官网教程部署即可。官网链接：https://1panel.cn/
基础服务配置，包括ssl证书等，其前提是先备案域名，我这里没申请成功，就算了。

# 4.个人博客Web端部署
本地开发静态博客，我使用react+tailwind+md的框架。  
我的操作流程：  
本地安装nodejs环境  
使用vite快速创建react+typescript环境  
安装tailwindcss和相关包，配置
编写md和渲染md的代码  
使用vite打包开发好的源码  
使用scp将dist包发送到服务器上  
在服务器上安装nginx框架，配置  
源码仓库https://github.com/htkhgsj/jufazhishang
